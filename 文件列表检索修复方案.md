# 文件列表检索修复方案

## 问题摘要

另一台机器上 WebChat 文件列表无法检索到状态目录中的 `.md` 等文件，表面症状像是 `stateDir` 没连上，但实际主故障链路是：

1. 启动器预先注入了 `BELLDANDY_AUTH_MODE=token` 与 `BELLDANDY_AUTH_TOKEN`。
2. Gateway 启动时使用了一份本地 `.env` 加载逻辑；该逻辑遇到已有进程环境变量会直接跳过，不会用 `.env` / `.env.local` 覆盖。
3. 因此配置文件中显式写的 `BELLDANDY_AUTH_MODE=none` 没有生效。
4. WebChat 握手未携带匹配 token 时，WebSocket 被 4403 拒绝，后续 `workspace.list` 无法拿到返回。

## 修复目标

1. 让 Gateway 与 CLI 复用同一份 `.env` 加载器，消除重复实现和行为漂移。
2. 明确统一配置优先级：
   - `.env` 覆盖启动器/外部 shell 预设值
   - `.env.local` 再覆盖 `.env`
3. 保持启动器注释与实际行为一致。
4. 为本次问题补回归测试，防止 token 预设再次压住文件配置。

## 实施方案

### 方案 1：统一复用共享 env loader（本次采用）

- 在 `packages/belldandy-core/src/cli/shared/env-loader.ts` 中补一个“按优先级加载 `.env` / `.env.local`”的共享入口。
- `packages/belldandy-core/src/bin/gateway.ts` 改为调用共享 loader，删除其内部重复的 `loadEnvFileIfExists` 副本。
- 加载顺序统一为：先 `.env`，后 `.env.local`。
- `packages/belldandy-core/src/cli/shared/context.ts` 与 `packages/belldandy-core/src/cli/commands/doctor.ts` 同步改为相同顺序，避免 CLI 与 Gateway 读取结果不一致。

### 方案 2：只改启动器（不采用）

- 让 `start.bat` / `start.sh` 仅在配置文件未显式设置鉴权时才注入 token。

不采用原因：

- 只能修一条启动路径；
- Gateway 内部重复实现仍然存在；
- CLI / 便携版 / 其他入口仍可能继续漂移。

## 影响范围

- `packages/belldandy-core/src/cli/shared/env-loader.ts`
- `packages/belldandy-core/src/bin/gateway.ts`
- `packages/belldandy-core/src/cli/shared/context.ts`
- `packages/belldandy-core/src/cli/commands/doctor.ts`
- 新增回归测试文件

## 风险点

1. 现有少数依赖“shell env 高于文件配置”的启动方式会改变行为。
2. `.env` 与 `.env.local` 的覆盖顺序修正后，若有人曾反向依赖旧行为，结果会变化。
3. 需要确认测试过程不会污染全局 `process.env`。

## 回滚方式

- 回滚上述几个文件的本次修改即可。
- 若仅测试失败但功能逻辑已确定，可先保留共享 loader 改动，单独回滚测试文件再继续排查。

## 测试清单

### 自动化回归测试

1. **`.env` 覆盖启动器预设 token**
   - 预置 `process.env.BELLDANDY_AUTH_MODE=token`
   - 预置 `process.env.BELLDANDY_AUTH_TOKEN=setup-token`
   - `.env` 写入 `BELLDANDY_AUTH_MODE=none`
   - `.env` 写入 `BELLDANDY_AUTH_TOKEN=file-token`
   - 断言加载后以文件值为准

2. **`.env.local` 覆盖 `.env`**
   - `.env` 写入 `BELLDANDY_AUTH_MODE=token`
   - `.env.local` 写入 `BELLDANDY_AUTH_MODE=none`
   - 断言最终为 `none`

3. **带引号值会被正确去引号**
   - `.env` / `.env.local` 写入带单双引号的 token
   - 断言载入后值已标准化

### 手动验证

1. 在 `.env` 中设置：
   - `BELLDANDY_AUTH_MODE=none`
2. 通过 `start.bat` 启动。
3. 确认 Gateway 运行态不再被强制压成 token 模式。
4. 打开 WebChat，确认文件树能正常拿到 `workspace.list` 返回。
5. 确认状态目录中的 `.md/.json/.txt` 能正常显示。

## 技术债决策

- `fix_now`：统一 env loader 与覆盖顺序，属于本次根因链路，直接处理。
- `defer`：启动器与便携版其余历史入口的进一步收敛，可在本次修复完成后另开整理任务。
