# Star Sanctuary 项目使用指南 (小白版)

欢迎使用 Star Sanctuary！这是一个运行在你本地电脑上的 **个人 AI 助手**。
它不像 ChatGPT 那样运行在别人的服务器上，而是完全属于你——注重隐私、安全，并且拥有记忆，能帮你操作电脑、上网查资料。

本指南专为非技术背景用户编写，略项了复杂的代码细节，助你 3 分钟快速上手。

## 首页速览

如果你只想先知道“我该从哪里开始”，看这张表就够了：

| 你现在要做什么 | 最优先入口 | 一句话说明 |
|---|---|---|
| 和 AI 正常聊天 | WebChat 聊天界面 | 日常使用直接在网页里发消息即可。 |
| 改模型、改 API Key、开关能力 | WebChat `⚙️ 设置` | 普通用户优先用可视化设置，不必先改配置文件。 |

| 看记忆、任务记录、经验候选 | 左侧 `🧠 记忆查看` | 适合看“它为什么这么答、之前做过什么”。 |
| 管理长期任务 | 左侧 `🎯 长期任务` | 适合分阶段推进长期事项。 |
| 用画布拆任务、看过程 | 左侧 `画布工作区` | 适合把复杂任务画出来、连起来。 |
| 临时禁用某些工具 / MCP / 插件 | 顶部 `🛠️ 工具设置` | 这是运行时临时禁用，不等于卸载。 |
| 看系统是否正常 | `corepack pnpm bdd doctor` | 普通检查优先这个命令。 |
| 安装 / 更新 / 卸载扩展 | `bdd marketplace ...` | 当前扩展安装主入口仍是命令行。 |

如果你不是开发者，可以先记住这个最简判断：

- 想直接用功能：优先 `WebChat`
- 想改配置或装扩展：优先 `CLI`
- 只有在做集成或排障时，才需要接触 `RPC / HTTP`

## 一、3 分钟快速上手

### 1. 下载与安装

首先，请确保你的电脑上安装了 **Node.js** (推荐版本 22+)。
[👉 点击下载 Node.js](https://nodejs.org/) (安装时一路点击 "Next" 即可)

然后，将 Star Sanctuary 项目文件夹下载到你的电脑上。

### 2. 一键启动

不需要敲复杂的命令。

- **Windows 用户**：双击项目目录下的 `start.bat` 文件。
- **Mac / Linux 用户**：在终端运行 `./start.sh`。

启动后，会发生两件事：
1.  弹出一个黑色的终端窗口（请不要关闭它，这是 Star Sanctuary 的"大脑"）。
2.  你的浏览器会自动打开 `http://localhost:28889`，进入聊天界面。

### 3. 首次配对 (重要安全步骤)

为了防止邻居或家里的猫咪连上你的 AI，首次使用时需要配对：

1.  在聊天界面发送第一条消息（比如 "你好"）。
2.  界面会弹出提示，显示一个 8 位配对码，例如：`Pairing required. Code: ABC123XY`。
3.  **另开一个新的终端窗口**（不要关掉那个黑色启动窗口！），进入 Star Sanctuary 项目所在的文件夹，执行以下命令：

    ```bash
    # 先进入 Star Sanctuary 项目目录（路径根据你的实际情况替换）
    cd D:\gongzuo\xiangmu\Belldandy\Belldandy

    # 执行配对批准命令，把 ABC123XY 换成界面上显示的实际配对码
    corepack pnpm bdd pairing approve ABC123XY
    ```

    > **💡 小贴士**：
    > - **项目目录**是指包含 `start.bat` 或 `package.json` 文件的那个文件夹。
    > - **配对码**在每次未配对设备发送消息时都会显示在聊天界面中。
    > - 配对码有效期 **1 小时**，超时需重新在聊天界面发一条消息获取新码。

4.  终端显示 `✓ Client xxx approved` 即配对成功。
5.  回到聊天界面再发一条消息，现在你可以和 Star Sanctuary 自由对话了！

## 二、设置你的 AI (让它能用)

Star Sanctuary 需要连接 AI 模型（如 OpenAI、Kimi、DeepSeek 等）才能变聪明。

### 方式一：可视化设置 (推荐 ✨)

这是最简单的方法：

1.  启动 Star Sanctuary 并打开聊天界面。
2.  点击右上角的 **⚙️ 设置图标**。
3.  在弹出的面板中填入你的 AI 服务信息：
    *   **Base URL**: 服务商的接口地址 (例如 `https://api.openai.com/v1` 或 `https://api.moonshot.cn/v1`)
    *   **API Key**: 你的密钥 (sk-xxxx...)
    *   **Model**: 模型名称 (例如 `gpt-4o` 或 `kimi-k2.5`)
4.  点击 **Save**。系统会自动重启，配置即刻生效。

### 方式二：配置文件 (进阶)

如果你更喜欢改文件，可以在项目根目录创建一个名为 `.env.local` 的文件，填入以下内容：

```env
# AI 服务商配置
BELLDANDY_OPENAI_BASE_URL=https://api.moonshot.cn/v1
BELLDANDY_OPENAI_API_KEY=sk-your-key-here
BELLDANDY_OPENAI_MODEL=kimi-k2.5

# 启用所有好用的功能
BELLDANDY_TOOLS_ENABLED=true      # 允许它使用工具
BELLDANDY_HEARTBEAT_ENABLED=true  # 允许它主动找你
```

## 三、核心功能：怎么玩？

### 💬 像微信一样聊天
- 直接在输入框打字即可。
- 支持 **Markdown**：它可以发表情包、写代码、列计划，排版很漂亮。

### 🕒 消息时间与最新标记
- WebChat 里每一条用户消息和 Agent 消息都会显示本地时间，方便你回看对话发生在什么时候。
- 当前会话最后一条消息会带一个 `最新` 标记；当新消息到来时，这个标记会自动移动到新的最后一条消息上。
- 刷新页面、切换会话或重新连接后，系统会按会话里的最后一条消息自动恢复 `最新` 标记。
- 这些时间信息也会一起提供给 Agent 作为上下文锚点，所以它更容易分辨“最近记忆”和“较早的历史记忆”。

### 🗣️ 会听会说 (语音交互)
- **听**：点击输入框右侧的 **🎤 麦克风** 图标，直接对它说话，它能听懂中文。
- **说**：告诉它 **"开启语音模式"**，它回复时就会读给你听（默认使用微软晓晓，声音很好听）。

### 👁️ 会看 (视觉感知)
- **看图片/视频**：点击输入框左侧的 **+ 号**，上传图片或视频文件。问它 "这图里有什么？" 或 "总结一下这个视频"。
    *   *注：需要使用支持视觉的模型，如 Kimi k2.5*。
- **看你屏幕**：(需安装浏览器插件) 对它说 **"看看我在干嘛"**，它会这截取你当前浏览器的画面并分析。

### 🌏 会上网 (浏览器操控)
Star Sanctuary 可以像人一样操作浏览器。

**1. 安装插件 (仅需一次)**：
- 打开 Chrome 浏览器 -> 扩展程序 -> 打开右上角"开发者模式"。
- 点击"加载已解压的扩展程序"，选择项目里的 `apps/browser-extension` 文件夹。

**2. 连接**：
- 点击浏览器右上角的 **B** 图标 (或紫色小幽灵)，显示 "Connected" 即连接成功。

**3. 使用**：
- 对它说：**"打开 B 站搜索猫咪视频"** 或 **"项 Google 查一下今天的天气"**。

## 五、高级玩法：让它帮你打工

### 🖥️ 后台运行 (Daemon 模式)
默认情况下，关掉那个黑色终端窗口，Star Sanctuary 就停止了。如果你想让它在后台默默运行（比如部署到服务器上），可以使用后台模式。

**跨平台说明**：

| 平台 | 行为 |
|------|------|
| Linux / macOS | 标准 daemon 进程，关闭终端后继续运行 |
| Windows | detached 模式可工作，但关闭终端窗口后进程可能被终止；生产部署建议使用 Windows Service 或 NSSM |

**启动后台运行**：
```bash
# 进入项目目录，例如：
cd E:\project\belldandy

# 后台启动
corepack pnpm bdd start -d
```

启动成功后会显示：
```
✓ Gateway started in background (PID 12345)
  Log file: ~/.star_sanctuary/logs/gateway.log
  Stop with: bdd stop
```

**查看运行状态**：
```bash
corepack pnpm bdd status
```

**停止后台运行**：
```bash
corepack pnpm bdd stop
```

> **💡 小贴士**：
> - 后台运行时，日志会写入 `~/.star_sanctuary/logs/gateway.log` 文件。
> - 普通使用建议直接双击 `start.bat`，更直观。后台模式适合服务器部署或长期运行场景。

### ⏰ 定时提醒 (Cron)
让它变成你的私人秘书。直接对它说：
- **"每天早上 9 点提醒我查看股市"**
- **"下午 3 点提醒我喝水"**
- **"列出所有定时任务"**

### 🎭 多重分身 (Multi-Agent)
点击聊天界面顶部的下拉菜单，可以切换不同的分身：
- **Default**: 全能助手
- **Coder**: 专门写代码
- **Researcher**: 专门查资料

### 🤖 指挥分身干活
你可以像老板一样，把任务分派给不同的分身。
对它说：**"让 Coder 写一个贪吃蛇游戏，同时让 Researcher 查一下最新的游戏设计趋势。"**
它会自动调度两个分身并行工作，最后给你汇总结果。

## 六、常见问题 (FAQ)

**Q: 启动不了，提示端口被占用？**
A: 说明 28889 端口被占了。项 `.env.local` 文件里加一行 `BELLDANDY_PORT=28890` 换个端口试试。

**Q: 我改了配置，怎么生效？**
A: 直接对它说 **"重启服务"**。界面会出现 3 秒倒计时，然后自动重启刷新。

**Q: 它能读我的本地文件吗？**
A: 可以。把文件拖进聊天框，或者告诉它文件路径 "读取 E:\docs\note.txt"。
