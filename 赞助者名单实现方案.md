# 自动化赞助者徽章实现方案

为了后续随着赞助者的增加，能够优雅、自动地在 README 中展示赞助者头像聚合图（SVG），避免频繁手动提交 README 更新，建议采用基于 **GitHub Actions + SVG 生成脚本** 的自动化方案。

本方案以常用的自动化开源赞助人徽章工具（如 [sponsors-svg](https://github.com/antfu/sponsors-svg) 或 [sponsors-api](https://github.com/sponsors-api/sponsors-api) 的思想）为基础。下面演示一个无需自己搭服务器，单凭 GitHub Actions 即可每日更新 SVG 并推送到特定分支或当前分支的实现流程。

---

## 方案 1：使用 `antfu/sponsors-svg`（针对使用 GitHub Sponsors）

如果未来你申请开通了 GitHub Sponsors 功能，这是目前很多前端大神在用的高颜值方案。它会读取你的 Sponsors 数据，并生成网格分布的头像墙 SVG 文件。

### 1. 准备工作
- 你的账号必须已经开启 GitHub Sponsors（或者可以配置 OpenCollective 等兼容数据源）。
- 获得一个具有读取赞助者权限的 **GitHub Personal Access Token (PAT)**，并将其添加到项目仓库的 `Settings -> Secrets and variables -> Actions` 中，命名为 `SPONSORS_TOKEN`。

### 2. 配置 GitHub Actions 工作流
在项目中新建文件 `.github/workflows/sponsors.yml`，内容如下：

```yaml
name: Generate Sponsors SVG

on:
  workflow_dispatch: # 允许手动触发
  schedule:
    - cron: "0 0 * * *" # 每天运行一次
  push:
    branches: [ main ]

jobs:
  generate-sponsors:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Generate Sponsors
        uses: antfu/sponsors-action@v3
        with:
          # 提供有权限读取你 github sponsors 数据的 PAT
          token: ${{ secrets.SPONSORS_TOKEN }}
          file: sponsors.svg
          fallback: true
          
      - name: Commit Action
        uses: stefanzweifel/git-auto-commit-action@v4
        with:
          commit_message: "chore: update sponsors svg"
          branch: main
          file_pattern: sponsors.svg
```

### 3. README.md 结构引入
一旦脚本跑通，在你的代码根目录就会自动生成 `sponsors.svg`，然后在你的 `README.md` 中的赞助名单章节里，直接用 `![Sponsors](./sponsors.svg)` 引用即可：

```markdown
## 💖 赞助者名单 (Sponsors)

非常感谢以下赞助者对 Star Sanctuary 的支持与信任！

<p align="center">
  <img src="./sponsors.svg" alt="Sponsors" />
</p>
```

## 方案 2：自定义 JSON + 自动生成脚本（适用国内爱发电 / 微信 / 支付宝场景）

很多国内的赞助并不走 GitHub Sponsors 这个平台，而是来自“爱发电”或者自己收款码。没法通过官方 API 直接抓取数据的情况下，我们可以退而求其次：**维护一个 JSON 数据文件，结合简单的脚本生成 Markdown 列表或者 SVG，并由 CI/CD 自动更新**。

### 1. 准备数据文件 `sponsors.json`
在项目根目录（或 `docs/` 目录）建立一个 JSON 文件，手动添加新赞助者：

```json
[
  {
    "name": "王道",
    "avatar": "https://avatars.githubusercontent.com/u/123456?v=4", 
    "link": "https://github.com/wangdao",
    "amount": "特别赞助",
    "isFirst": true
  }
]
```
*(如果有些赞助者没有头像，也可以用默认头像替代，或者干脆只渲染名字网络)*

### 2. 编写构建徽章的脚本
你可以用 Node.js 写一个极其轻量的小脚本，比如 `scripts/generate-sponsors.js`：

```javascript
import fs from 'fs';

// 读取 JSON
const sponsors = JSON.parse(fs.readFileSync('./sponsors.json', 'utf-8'));

let markdown = `## 💖 赞助者名单 (Sponsors)\n\n非常感谢所有支持和赞助 Star Sanctuary 项目的朋友们，是你们的帮助让项目能够变得更好！\n\n`;

// 找出包含 `isFirst` 属性的赞助者，或者是按赞助等级排列
const firstSponsor = sponsors.find(s => s.isFirst);
if (firstSponsor) {
    markdown += `### 🥇 第一位特别赞助者\n\n**特别鸣谢：${firstSponsor.name}**\n您是本项目历史上的**第一位赞助者**！这份从 0 到 1 的信任与支持对我们具有极大的里程碑意义，万分感谢！\n\n---\n\n`;
}

// 通过 HTML 的 img 和 a 标签拼接头像墙
markdown += `### 💎 其他赞助者支持\n\n`;
markdown += '<p align="center">\n';
sponsors.filter(s => !s.isFirst).forEach(sponsor => {
    // 若不存在链接或头像，可以设默认值
    const link = sponsor.link || '#';
    const avatar = sponsor.avatar || 'https://ui-avatars.com/api/?name=' + encodeURIComponent(sponsor.name) + '&background=random';
    
    markdown += `  <a href="${link}" target="_blank">\n`;
    markdown += `    <img src="${avatar}" width="50" height="50" alt="${sponsor.name}" title="${sponsor.name} (${sponsor.amount})"\n`;
    markdown += `         style="border-radius: 50%; margin: 5px;" />\n`;
    markdown += `  </a>\n`;
});
markdown += '</p>\n';

// 我们可以用这个脚本动态去替换 README 的某两个特定标记 <!-- sponsors_start --> 和 <!-- sponsors_end --> 里的内容
const readmePath = './README.md';
let readmeContent = fs.readFileSync(readmePath, 'utf-8');

const regex = /<!-- sponsors_start -->[\s\S]*<!-- sponsors_end -->/;
const replacement = `<!-- sponsors_start -->\n\n${markdown}\n\n<!-- sponsors_end -->`;

if (readmeContent.match(regex)) {
    readmeContent = readmeContent.replace(regex, replacement);
    fs.writeFileSync(readmePath, readmeContent);
    console.log("README.md sponsors section updated!");
} else {
    console.log("Could not find the <!-- sponsors_start --> tags in README.md");
}
```

### 3. 修改 `README.md`
在 `README.md` 中为你打算注入内容的地方打上注释占位符：

```markdown
<!-- sponsors_start -->
<!-- sponsors_end -->
```

### 4. 设置 GitHub Actions
只要你在 `main` 分支修改了 `sponsors.json` 文件并推送到 GitHub，由 CI 去触发上面的那个 JS 脚本，最后由 CI 把组装好的 `README.md` 重新提交，就能完成页面自动化编排：

```yaml
name: Generate Sponsors Display

on:
  push:
    paths:
      - 'sponsors.json'
      - 'scripts/generate-sponsors.js'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: 18

      - name: Run generation script
        run: node scripts/generate-sponsors.js

      - name: Commit and push changes
        uses: stefanzweifel/git-auto-commit-action@v4
        with:
          commit_message: "docs(sponsors): auto-update sponsors section"
          file_pattern: README.md
```

### 总结比较
*   **方案 1（纯静态 SVG）：** 视觉效果最好，最现代感，但**高度依赖正规的第三方平台**和提供好的 API 爬虫（针对国内个人收款方式扩展性比较弱）。
*   **方案 2（自动化注入 README）：** 最灵活，自己只需维护一个 `sponsors.json` 或 `csv` 数据库就行，不管你是从支付宝、微信还是爱发电收到的赞助，自己添加一条记录在 JSON 里面，提交后 CI 就会帮你排好版。这也是针对独立开发者**强烈推荐**的做法。
