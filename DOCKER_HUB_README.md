# Star Sanctuary - Local-first Personal AI Assistant

[![Docker Pulls](https://img.shields.io/docker/pulls/vrboyzero/star-sanctuary)](https://hub.docker.com/r/vrboyzero/star-sanctuary)
[![Docker Image Size](https://img.shields.io/docker/image-size/vrboyzero/star-sanctuary/latest)](https://hub.docker.com/r/vrboyzero/star-sanctuary)
[![GitHub](https://img.shields.io/github/license/vrboyzero/star-sanctuary)](https://github.com/vrboyzero/star-sanctuary)

Star Sanctuary 是一个本地优先的个人 AI 助手。这个镜像适合通过 Docker / Docker Compose 在本机、NAS 或轻量服务器上快速部署。

当前兼容策略：

- 对外品牌、仓库与镜像统一为 `Star Sanctuary`
- 运行时环境变量仍保留 `BELLDANDY_*`
- Compose service key 仍保留 `belldandy-gateway` / `belldandy-cli`

## Highlights

- Local-first: state, logs, sessions, memory and methods stay on your own storage.
- Built-in tools: file ops, patching, web fetch, browser automation, memory retrieval and more.
- Secure by default: pairing + token/password auth, optional Origin allowlist.
- Multi-channel: WebChat first, with Feishu/Lark and other channels available in the codebase.
- Multi-arch images: `linux/amd64` and `linux/arm64`.

## Quick Start

### Docker Compose

```bash
curl -O https://raw.githubusercontent.com/vrboyzero/star-sanctuary/main/docker-compose.yml
curl -O https://raw.githubusercontent.com/vrboyzero/star-sanctuary/main/.env.example

cp .env.example .env
```

Edit `.env` and set at least:

```env
BELLDANDY_IMAGE=vrboyzero/star-sanctuary:latest
BELLDANDY_AUTH_MODE=token
BELLDANDY_AUTH_TOKEN=your-secure-token
BELLDANDY_AGENT_PROVIDER=openai
BELLDANDY_OPENAI_BASE_URL=https://api.openai.com/v1
BELLDANDY_OPENAI_API_KEY=sk-your-api-key
BELLDANDY_OPENAI_MODEL=gpt-4o
```

Start the gateway:

```bash
docker compose up -d belldandy-gateway
docker compose logs -f belldandy-gateway
```

Open <http://localhost:28889>.

### docker run

```bash
docker run -d \
  --name star-sanctuary \
  -p 28889:28889 \
  -v ~/.star_sanctuary:/home/belldandy/.star_sanctuary \
  -e BELLDANDY_AUTH_MODE=token \
  -e BELLDANDY_AUTH_TOKEN=your-secure-token \
  -e BELLDANDY_AGENT_PROVIDER=openai \
  -e BELLDANDY_OPENAI_BASE_URL=https://api.openai.com/v1 \
  -e BELLDANDY_OPENAI_API_KEY=sk-your-api-key \
  -e BELLDANDY_OPENAI_MODEL=gpt-4o \
  -e BELLDANDY_TOOLS_ENABLED=true \
  vrboyzero/star-sanctuary:latest
```

## Required Environment Variables

| Variable | Purpose | Example |
|---|---|---|
| `BELLDANDY_AUTH_MODE` | Auth mode: `none` / `token` / `password` | `token` |
| `BELLDANDY_AUTH_TOKEN` | Token for gateway auth | `openssl rand -hex 32` |
| `BELLDANDY_AGENT_PROVIDER` | Provider type | `openai` |
| `BELLDANDY_OPENAI_BASE_URL` | OpenAI-compatible API base URL | `https://api.openai.com/v1` |
| `BELLDANDY_OPENAI_API_KEY` | Provider API key | `sk-...` |
| `BELLDANDY_OPENAI_MODEL` | Model name | `gpt-4o` |

Useful optional variables:

| Variable | Default | Notes |
|---|---|---|
| `BELLDANDY_IMAGE` | `vrboyzero/star-sanctuary:local` | Compose image tag override |
| `BELLDANDY_HOST` | `127.0.0.1` | Set `0.0.0.0` for LAN access |
| `BELLDANDY_GATEWAY_PORT` | `28889` | Published port on the host |
| `BELLDANDY_STATE_DIR` | `~/.star_sanctuary` | Host-side state path; absolute path recommended |
| `BELLDANDY_WORKSPACE_DIR` | `./workspace` | Optional workspace bind mount |
| `BELLDANDY_TOOLS_ENABLED` | `true` | Enable tool calling |
| `BELLDANDY_MEMORY_ENABLED` | `true` | Enable memory system |
| `BELLDANDY_LOG_LEVEL` | `info` | Runtime log level |

Full template: <https://github.com/vrboyzero/star-sanctuary/blob/main/.env.example>

## Data Persistence

Container state is stored at:

```text
/home/belldandy/.star_sanctuary
```

Typical host mount:

```yaml
volumes:
  - /absolute/path/to/star_sanctuary:/home/belldandy/.star_sanctuary
  - /absolute/path/to/workspace:/home/belldandy/workspace
```

Stored data includes:

- `SOUL.md`, `IDENTITY.md`, `USER.md`
- `memory.db`
- `logs/`
- `sessions/`
- `methods/`
- `allowlist.json`

## Tags

- `latest`: latest stable image published from the default branch / release pipeline.
- `0.1.0`: fixed release version.
- `0.1`: moving minor tag.
- `0`: moving major tag.
- `main-<sha>`: branch / SHA build tag generated by CI metadata rules.

## Upgrade

Use fixed tags in production:

```bash
docker pull vrboyzero/star-sanctuary:0.1.0
docker compose up -d --remove-orphans
```

Or follow `latest` in test environments:

```bash
docker pull vrboyzero/star-sanctuary:latest
docker compose up -d --remove-orphans
```

## Troubleshooting

Container failed to start:

```bash
docker compose logs belldandy-gateway
```

Check:

1. `BELLDANDY_AUTH_TOKEN`, `BELLDANDY_OPENAI_API_KEY`, `BELLDANDY_OPENAI_MODEL`
2. `BELLDANDY_STATE_DIR` is a valid absolute path on the host
3. Port `28889` is not already occupied

Health check failed:

```bash
docker exec -it belldandy-gateway bash
curl http://127.0.0.1:28889/health
```
