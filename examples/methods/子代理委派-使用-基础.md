# 子代理使用指南

## 概述

子代理（Sub-Agent）是一种将复杂任务委托给独立代理执行的机制。子代理在独立上下文中运行，但共享工作区资源。

## 重要：配置文件位置

> **⚠️ 纠正**：子代理配置文件位于 `~/.star_sanctuary/agents.json`，**不是** `agents/` 目录！

```
~/.star_sanctuary/
├── agents.json          # ← 配置文件在这里！（根目录）
├── agents/             # ← Agent 专属工作区目录（不是配置文件）
│   ├── coder/
│   │   └── SOUL.md     # coder 的自定义人格
│   └── researcher/
│       └── SOUL.md     # researcher 的自定义人格
├── SOUL.md             # default Agent 的人格
└── ...
```

## 可用工具

| 工具 | 功能 | 适用场景 |
|------|------|----------|
| `delegate_task` | 单任务委托 | 精准投递单个复杂任务 |
| `delegate_parallel` | 并行委托 | 多个独立任务同时处理 |
| `sessions_spawn` | 生成子代理 | 独立新任务，全新上下文 |
| `sessions_history` | 查看历史 | 监控子代理状态和历史 |

## 可用子代理

| agent_id | 说明 | 状态 | 工具权限 |
|----------|------|------|----------|
| `default` | 默认子代理 | ✅ 可用 | 全部工具 |
| `coder` | 编码专家 | ✅ 已配置 | file_read, file_write, run_command 等 |
| `researcher` | 研究专家 | ✅ 已配置 | web_fetch, web_search, memory_search 等 |

## 工具参数

### delegate_task
```json
{
  "agent_id": "coder",
  "instruction": "详细指令",
  "context": { "cwd": "工作目录", "其他": "上下文" }
}
```

### delegate_parallel
```json
{
  "tasks": [
    { "instruction": "任务1", "agent_id": "coder", "context": {} },
    { "instruction": "任务2", "agent_id": "researcher", "context": {} }
  ]
}
```

### sessions_spawn
```json
{
  "instruction": "任务指令",
  "context": { "cwd": "工作目录" }
}
```

## 使用示例

### 单任务委托（coder）
```bash
delegate_task agent_id="coder" instruction="列出工作区根目录文件"
```

### 并行任务（混合代理）
```bash
delegate_parallel tasks=[
  {"instruction": "列出工作区文件", "agent_id": "coder"},
  {"instruction": "调研AI技术趋势", "agent_id": "researcher"}
]
```

### 任务状态查询
```bash
sessions_history
```

## 编排策略

### 任务分配原则

| 任务类型 | 推荐代理 | 原因 |
|----------|----------|------|
| 代码开发/修改/调试 | `coder` | 专业编码能力，工具权限包含 file 系列 |
| 调研/分析/信息收集 | `researcher` | 研究能力，工具权限包含 web_search |
| 通用任务 | `default` | 万能型，拥有全部工具权限 |

### 并行编排模式

1. **同构并行**：相同代理处理多个相似任务
   - 示例：多个 coder 同时处理不同文件的代码审查

2. **异构并行**：不同代理处理不同领域任务
   - 示例：coder + researcher 组合，一个写代码一个调研

3. **流水线**：任务有依赖关系时，串行执行
   - 示例：researcher 调研 → coder 根据调研结果开发

## 成功案例

### 案例1：单代理测试
- **代理**：`coder`
- **任务**：列出工作区文件并统计
- **结果**：✅ 成功返回 11 文件夹 + 21 文件

### 案例2：并行异构任务
- **代理**：`coder` + `researcher`
- **任务**：
  - coder：列出工作区根目录文件
  - researcher：调研2026年AI技术趋势
- **结果**：✅ 两个任务同时完成，结果聚合

## 失败经验

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| AgentProfile not found | agent_id 不存在 | 确认 agents.json 中已配置 |
| 任务超时 | 指令太复杂 | 拆分为更小的任务 |
| 上下文丢失 | context 参数缺失 | 明确传递必要的上下文 |
| 配置不生效 | 修改后未重启服务 | 修改 agents.json 后需重启服务 |
| 配置文件找不到 | 在 agents/ 目录下找 | 配置文件在 ~/.star_sanctuary/agents.json |
| 工具权限不足 | agent 的 toolWhitelist 未包含 | 在 agents.json 中添加工具到 toolWhitelist |

## 注意事项

1. **独立上下文**：子代理有独立推理上下文，不干扰主会话
2. **共享工作区**：子代理可以访问当前工作区文件
3. **结果聚合**：`delegate_parallel` 会汇总所有子代理结果
4. **重启生效**：修改配置后需要重启服务
5. **会话追踪**：`sessions_history` 返回的 ID 可用于追踪任务
6. **专属工作区**：非 default 的 Agent 可有独立人格文件，位于 `agents/{id}/`
