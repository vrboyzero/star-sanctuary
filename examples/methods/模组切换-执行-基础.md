## 适用场景

- 用户要求切换 FACET 模组（如"切换模组为 [文件名]" 或 "切换 FACET 为 [文件名]"）
- 需要更换当前 Agent 的角色定位和行为模式
- 从 facets/ 目录加载新的职能模组

---

## 执行步骤（v2.0 - 工具化流程）

### Step 1: 确认模组文件存在

1. 确认 facets 目录位置：`~/.belldandy/facets/`
2. 检索目标文件：`[文件名].md`
3. **验证文件存在**：使用 `list_files` 或 `file_read` 确认文件可读

### Step 2: 调用 switch_facet 工具

**核心工具**：`switch_facet`

**参数**：
- `facet_name`: 模组文件名（**不含 `.md` 后缀**）

**执行逻辑**：
1. 工具会自动完成 SOUL.md 的锚点定位
2. 工具会自动完成内容替换和原子写入
3. 工具会返回切换结果（成功/失败及原因）

### Step 3: 确认切换结果

**检查工具返回**：
- 如果返回成功：继续下一步
- 如果返回失败（如模组不存在）：向用户反馈具体原因

### Step 4: 环境纯化（必须执行）

**调用 `service_restart` 工具**
- 重启服务以清空旧模组的推理惯性和语气干扰
- 等待重启完成

### Step 5: 冷启动初始化

重启后，按【SYSTEM | 身份定义 | 核心机能】中的"开场白规范"执行：

1. **身份宣告**：汇报当前激活的 [FACET] 模组身份
2. **进度检索**：简述从 `memory/` 或向量检索中找回的最后任务进度
3. **主动引导**：询问拍档下一步的指令或建议

## 失败经验

### 失败案例 #1：章节标题不匹配导致双章节（v1.0 时代）

| 属性 | 详情 |
|------|------|
| **发生时间** | 2026-02-11 |
| **失败场景** | 从默认模组切换到"铁血领袖"模组 |
| **失败原因** | 旧章节标题 `## 【FACET \| 职能模组】` 与新章节标题 `## 【FACET \| 模组 \| 铁血领袖】` 不匹配，导致替换失败 |
| **错误表现** | SOUL.md 中同时存在两个 FACET 章节，旧内容未被删除 |
| **根本原因** | 1. 章节标题格式不一致<br>2. 使用了简单的字符串替换而非全量覆盖<br>3. 缺乏替换后的验证步骤 |
| **解决方案** | 升级到 v2.0，使用 `switch_facet` 工具自动处理锚点定位和替换 |

### 失败案例 #2：未验证 facets 目录存在

| 属性 | 详情 |
|------|------|
| **发生时间** | 2026-02-11 |
| **失败原因** | 未主动使用 `list_files` 检查 facets 目录内容 |
| **解决方案** | 切换前执行 `list_files path="facets"` 确认目录状态 |

### 失败案例 #3：未执行 service_restart

| 属性 | 详情 |
|------|------|
| **发生时间** | 2026-02-11 |
| **失败场景** | 切换模组后未重启服务 |
| **失败原因** | 旧模组的推理惯性和语气残留 |
| **错误表现** | 新模组的行为模式未完全生效 |
| **解决方案** | **强制要求**：每次切换后必须调用 `service_restart` |
