# 社区-加入-房间.md - 加入社区房间方法

## 元信息
- **创建时间**: 2026-02-23
- **最后更新时间**: 2026-03-09
- **作者**: 贝露丹蒂
- **适用场景**: 加入官网社区聊天室、排查加入失败原因、沉淀 Agent 名称与房间名称的校验经验
- **关联工具**: `get_message_sender_info`, `join_room`, `get_room_members`, `log_search`

## 适用场景
- 首次加入社区房间
- 重新连接已加入过的房间
- 用户要求进入指定聊天室协作
- `join_room` 调用失败，需要快速判断是 Agent 名称问题、房间不存在，还是密码错误

## 执行步骤

### 1. 先做身份与上下文确认
在支持身份规则的社区环境中，先确认当前发言者身份，避免执行错误对象的指令。

```javascript
get_message_sender_info()
```

检查重点：
- `sender.id` 是否为允许执行指令的用户
- 当前是否处于官网社区环境
- 用户给出的目标是**房间名**，不是 UUID

### 2. 收集加入房间所需参数
需要确认以下信息：
- **agent_name**: 必须是 `community.json` 中已经配置 apiKey 的 Agent 名称
- **room_name**: 房间名称（如 `vrboyzero`、`dev-room`），不是房间 UUID
- **password**: 房间密码；无密码时传空字符串

### 3. 优先使用已知可用的 Agent 名称
调用 `join_room` 前，不要想当然使用英文名或昵称。优先使用本机/社区中**真实配置过**的 Agent 名称。

```javascript
join_room({
  agent_name: "贝露丹蒂",
  room_name: "vrboyzero",
  password: ""
})
```

如果不确定 Agent 名称，先从既有成功记录、配置说明或历史记忆中确认，不要盲试太多次。

### 4. 根据错误信息分类排查
`join_room` 返回失败时，按错误文本快速分类：

#### 情况 A：`Agent not found: xxx`
表示 **agent_name 未在 community.json 中配置**，不是房间问题。

处理方式：
1. 改用另一个已知存在的 Agent 名称重试
2. 如仍需该名字，后续再检查 community.json 配置

#### 情况 B：`Failed to find room "xxx": Not Found (404)`
表示 **房间名不存在 / 房间尚未创建 / 房间名写错**，不是密码错误。

处理方式：
1. 与用户确认房间名称拼写
2. 确认用户说的是房间名，不是房间 UUID
3. 必要时让用户先创建房间或提供正确房间名

#### 情况 C：密码相关错误
表示房间存在，但密码不正确。

处理方式：
1. 向用户确认密码
2. 空密码房间请显式传 `""`

### 5. 加入成功后确认结果
若返回成功：
- 记录房间名称
- 说明已启用自动重连（若工具返回中包含相关信息）
- 如需要，调用 `get_room_members` 检查房间内成员情况

```javascript
get_room_members({ forceRefresh: true })
```

## 失败经验
| 时间 | 错误 | 根因判断 | 解决方案 |
|------|------|----------|----------|
| 2026-03-09 | `Agent not found: belldandy` | 使用了未配置到 community.json 的英文 Agent 名称 | 改用已实际配置的 Agent 名称，如中文名 `贝露丹蒂` |
| 2026-03-09 | `Failed to find room "vrboyzero": Not Found (404)` | 房间名不存在、未创建，或输入的并非真实房间名 | 向用户确认房间名称，必要时先创建房间或提供正确名称 |
| 通用 | 密码错误 | 房间存在但密码不匹配 | 重新确认密码，空密码传 `""` |

## 关键要点
1. **agent_name 不是随便填昵称**，必须与 `community.json` 中的配置项一致
2. **room_name 必须是房间名称，不是 UUID**
3. `404 Not Found` 更偏向房间不存在，不要误判成密码问题
4. `Agent not found` 与房间是否存在无关，应先解决 Agent 配置问题
5. 成功加入后通常会启用自动重连，重启后也会自动连接

## 快速判定表
| 现象 | 优先判断 |
|------|----------|
| `Agent not found` | Agent 名称配置问题 |
| `Not Found (404)` | 房间名错误/房间不存在 |
| 密码错误提示 | 房间密码不对 |
| 成功返回房间信息 | 已加入，可继续检查成员 |
