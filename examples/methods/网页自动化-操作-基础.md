## 📖 标准操作流程（SOP）

### 阶段一：准备与连接

```
Step 1: 确认扩展已安装并连接
- 询问用户是否已安装扩展
- 确认用户已点击 "Attach Tab"
- 等待用户确认连接成功
```

### 阶段二：获取页面信息

**关键原则**：**必须先获取页面快照，了解页面结构后再操作！**

```bash
# 获取页面完整快照（查看所有可交互元素）
browser action=snapshot profile=chrome maxChars=50000
```

**快照信息包含**：
- 页面标题、URL
- 所有可交互元素的 ID 和描述
- 输入框、按钮、链接的位置信息

### 阶段三：执行操作

#### 3.1 导航到指定页面

```bash
# 导航到目标URL
browser action=navigate url="https://www.bilibili.com"
```

#### 3.2 点击元素

```bash
# 使用元素ID点击（推荐）
browser action=act profile=chrome request={"kind":"click","ref":"元素ID"}

# 示例：点击搜索按钮
browser action=act profile=chrome request={"kind":"click","ref":"nav_searchform"}
```

#### 3.3 输入文字

```bash
# 在输入框中输入文字
browser action=act profile=chrome request={"kind":"type","ref":"输入框ID","text":"要输入的内容"}

# 示例：在搜索框输入"周深"
browser action=act profile=chrome request={"kind":"type","ref":"search-keyword","text":"周深"}
```

#### 3.4 截图

```bash
# 截取当前页面
browser action=screenshot profile=chrome
```

### 阶段四：信息提取与分析

1. **从截图中识别关键信息**
2. **记录URL、标题、关键数据**
3. **整理成结构化信息返回给用户**

## ✅ 成功案例一：B站搜索周深歌曲

### 操作步骤

```
1. 确认用户已连接 Star Sanctuary Browser Relay
2. 导航到 B站搜索页面
   browser action=navigate url="https://search.bilibili.com/all?keyword=周深"

3. 获取页面快照，找到"卡布叻_周深"主页链接
   browser action=snapshot profile=chrome maxChars=50000

4. 从快照中识别出用户主页链接
   - 用户名：卡布叻_周深
   - UID：3404595
   - 链接：https://space.bilibili.com/3404595

5. 导航到用户主页
   browser action=navigate url="https://space.bilibili.com/3404595"

6. 获取主页快照，分析最新视频
   browser action=snapshot profile=chrome maxChars=50000

7. 从快照中提取最新两首歌曲信息：
   - 《奔腾》- 2026年1月5日发布
   - 《我的对》- 2025年12月31日发布

8. 整理信息并返回给用户
```

### 成功要素

| 要素 | 说明 |
|------|------|
| **先快照后操作** | 每次操作前都先获取页面快照，了解页面结构 |
| **直接使用 navigate** | 对于已知URL，直接用 navigate 跳转，而不是点击 |
| **信息结构化** | 从快照中提取关键信息，整理成表格/列表形式 |
| **分步执行** | 每个操作后检查结果，再执行下一步 |

## 📊 数据提取技巧

### 从快照中提取结构化数据

**原则**：
- 保留原始格式（如"327.0万"而非"3270000"）
- 识别数据单位（万、亿、K、M）
- 提取多维度数据（粉丝、获赞、播放等）
- 保持数字精准度，不做四舍五入

**示例**：
```
快照内容：
粉丝 327.0万
获赞 394.5万
播放 7145.3万

提取结果：
- 粉丝数：327.0万
- 获赞数：394.5万
- 播放数：7145.3万
```

## ❌ 失败经验与教训

### 失败案例一：直接点击搜索框

**错误操作**：
```bash
# 错误：没有先获取快照，直接尝试点击
browser action=act profile=chrome request={"kind":"click","ref":"搜索框"}
```

**失败原因**：
- 没有先获取页面快照，不知道正确的元素ID
- "搜索框"不是有效的元素引用标识

**正确做法**：
```bash
# 1. 先获取快照
browser action=snapshot profile=chrome maxChars=50000

# 2. 从快照中找到正确的元素ID（如 "nav_searchform"）
# 3. 使用正确的ID点击
browser action=act profile=chrome request={"kind":"click","ref":"nav_searchform"}
```

### 失败案例二：尝试使用 web_fetch 替代

**错误决策**：
- 当 Star Sanctuary Browser Relay 遇到问题时，尝试切换到 web_fetch
- 但 web_fetch 无法执行JavaScript，无法获取动态内容

**教训**：
- **坚持主工具**：Star Sanctuary Browser Relay 是首选，遇到问题先排查使用方法
- **不要频繁切换工具**：除非用户特别指定，否则应坚持解决当前工具的问题

### 失败案例三：操作超时未重试

**错误操作**：
- 点击或导航操作超时后，没有尝试其他方案
- 直接放弃并告知用户工具故障

**正确做法**：
- 操作超时后，先截图确认当前页面状态
- 尝试使用 navigate 直接跳转URL
- 或询问用户当前页面状态，调整策略

## 🔧 故障排查

### 问题一：操作超时

**现象**：click 或 type 操作超时

**解决方案**：
1. 先执行截图，确认当前页面状态
2. 尝试使用 navigate 直接跳转目标URL
3. 询问用户当前页面显示什么内容
4. 重新获取快照，检查元素ID是否变化

### 问题二：元素找不到

**现象**：提示元素不存在或无法点击

**解决方案**：
1. 重新获取页面快照
2. 检查元素ID是否正确
3. 确认页面是否已加载完成
4. 尝试使用其他定位方式（如文本内容）

### 问题三：扩展未连接

**现象**：所有操作都失败

**解决方案**：
1. 询问用户是否已安装扩展
2. 确认用户已点击 "Attach Tab"
3. 建议用户刷新页面后重新连接
