# MEMORY-query-recall.md - 记忆检索方法论

> 当用户要求"回忆"、"查找"、"看看之前"相关内容时的标准检索流程

## 📋 适用场景

- 用户说"还记得吗"、"之前我们聊过"、"上次那个"
- 用户要求回顾特定对话、任务、事件
- 在 memory/ 和 MEMORY.md 找不到相关内容时
- 需要检索历史对话记录

## 🎯 执行步骤

### 步骤 1：初步快速检查
先快速浏览核心记忆文件，建立基础上下文：
```
1. 读取 MEMORY.md（长期核心记忆）
2. 读取当日 memory/YYYY-MM-DD.md（如有）
```

### 步骤 2：向量语义检索（核心）
使用 `memory_search` 进行语义向量搜索：

```
memory_search
  query="[用户提到的关键词/事件]"
  memory_type="session"        # 优先检索对话记录
  detail_level="summary"       # 先看摘要
  limit=10
```

### 步骤 3：如果步骤2无结果，扩展检索
```
# 扩大范围检索
memory_search
  query="[关键词]"
  memory_type="daily,core,session"  # 全类型
  detail_level="summary"
  limit=15

# 缩小日期范围（如知道大致时间）
memory_search
  query="[关键词]"
  date_from="2026-02-01"
  date_to="2026-02-23"
```

### 步骤 4：深度读取
找到相关记录后，根据需要读取完整内容：
```
memory_search
  query="[关键词]"
  detail_level="full"    # 完整内容
  limit=3
```

### 步骤 5：反馈与确认
向用户反馈检索结果，包括：
- 找到的记录数量
- 相关程度评估
- 是否需要进一步查找

## 🔧 工具选择

| 工具 | 用途 | 优先级 |
|------|------|--------|
| `memory_search` | 向量语义检索（主要工具） | ⭐⭐⭐ |
| `file_read` | 读取 MEMORY.md、memory/*.md | ⭐⭐⭐ |
| `log_search` | 检索系统日志（极端情况） | ⭐ |

## ⚠️ 失败经验

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 检索不到 | 只搜索了 memory/ 目录，忽略 sessions | 必须包含 memory_type="session" |
| 检索不到 | query 太具体，匹配不到 | 提取核心关键词重新搜索 |
| 结果太多 | query 太宽泛 | 添加日期范围过滤 |
| 结果不相关 | memory_type 过滤错误 | 扩大为 all 或逐个尝试 |

## ✅ 成功案例

- **案例 1**：用户问"之前说的那个项目"
  - 检索关键词："项目"、"任务"、"对话"
  - 使用 `memory_type="session,daily"`
  - 成功找到相关对话记录

- **案例 2**：用户问"上周在社区房间的发言"
  - 检索关键词："社区"、"房间"、"vrboyzero"
  - 添加日期范围过滤
  - 成功定位到具体对话

## 📌 关键要点

1. **必须检索 sessions 对话记录**：不要只搜 memory/ 和 MEMORY.md
2. **先关键词后完整**：先用 summary 快速筛选，再按需读取 full
3. **灵活使用过滤**：日期范围、memory_type 都是有效过滤手段
4. **找不到时明确告知**：不要编造内容，如实反馈未找到

## 🔄 检索优先级总结

```
1. MEMORY.md（核心记忆）
       ↓ 无结果
2. memory/YYYY-MM-DD.md（当日记忆）
       ↓ 无结果
3. memory_search + memory_type="session"（对话记录）
       ↓ 无结果
4. 扩大范围：memory_type="all" + 日期过滤
       ↓ 仍无结果
5. 明确告知用户未找到，可建议补充关键词
```
