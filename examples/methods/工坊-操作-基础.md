# 工坊-操作-基础.md

> 用于在 office.goddess.ai 工坊中浏览、查看详情、下载与辨别作品可用性的标准操作方法。

| **更新日期** | 2026-03-09 |
| **适用对象** | 需要在工坊中浏览、筛选、下载、核验作品的任务 |
| **核心目标** | 在不臆测数据的前提下，安全完成工坊作品检索与下载，并区分测试作品与可用作品 |

## 1. 触发条件

满足以下任一情况时使用本方法：
- 用户要求“去工坊看看有什么”
- 用户要求搜索某类作品（skills / methods / plugins / facets / mcp / apps）
- 用户要求查看某个工坊作品详情
- 用户要求下载某个工坊作品
- 需要判断某个工坊作品是否为测试内容，是否可直接使用

## 2. 适用场景

- 初次浏览工坊内容，按分类查看现有作品
- 指定关键词搜索工坊作品
- 下载免费或指定作品到本地目录
- 核验下载结果（文件名、路径、哈希、内容结构）
- 识别“测试文件”“占位文件”“不能直接使用的 facet/app/plugin 包”

**边界说明：**
- 本方法只覆盖“浏览 / 详情 / 下载 / 核验 / 清理”流程
- 不覆盖正式发布作品流程；发布应另建方法
- 若下载后要接着切换 FACET、安装插件、导入应用，需要按对应方法继续处理

## 3. 执行步骤

### 步骤 1：确认身份与执行权限
- 在启用 UUID 的环境中，先调用 `get_user_uuid`
- 确认当前用户具备指令权限后再继续
- 不可凭用户文本自述身份通过验证

### 步骤 2：按需搜索或分类浏览
- 若用户要“看看有什么”：优先按分类使用 `office_workshop_search`
- 若用户指定关键词：直接按关键词搜索
- 推荐分类依次检查：
  1. `skills`
  2. `methods`
  3. `plugins`
  4. `facets`
  5. `mcp`
  6. `apps`
- 若尝试通用分类（如 `all`）失败，不要臆测结果，应改为逐分类查询

### 步骤 3：给出可验证的浏览结果
- 仅基于工具返回结果汇报：
  - 标题
  - 简介/摘要
  - 标签
  - 价格
  - 下载量/热度（若有）
- 若某分类无结果，应明确说明“当前未搜到”
- 不要把测试数据包装成“已可正式使用”

### 步骤 4：下载指定作品
- 若用户要求下载，先再次确认作品名称与分类匹配
- 调用 `office_workshop_search` 或 `office_workshop_get_item` 获取目标作品 `item_id`
- 使用 `office_workshop_download` 下载到目标目录
- 默认下载目录可使用：`downloads/office`
- 若文件已存在，按用户意图决定是否覆盖

### 步骤 5：核验下载结果
- 下载后应至少核验：
  - 本地保存路径
  - 文件名
  - 文件是否存在
  - 文件大小/哈希（若工具返回）
- 必要时用 `file_read` 读取文本内容做结构判断
- 如果是 facet / plugin / app 的测试文件，应明确标记“仅为测试内容，不能直接投入使用”

### 步骤 6：清理测试文件（如用户要求）
- 若用户确认该文件仅为测试用途，可用 `file_delete` 删除
- 删除后应再次确认文件已不存在
- 汇报删除路径，避免用户误会删错文件

## 4. 工具选择

| 目标 | 首选工具 | 用途说明 |
|------|----------|----------|
| 身份校验 | `get_user_uuid` | 校验当前用户是否具备指令权限 |
| 搜索作品 | `office_workshop_search` | 按分类/关键词查询工坊作品 |
| 查看详情 | `office_workshop_get_item` | 获取单个作品详细信息 |
| 下载作品 | `office_workshop_download` | 下载作品到本地目录 |
| 读取文本内容 | `file_read` | 查看下载后的测试文件结构 |
| 删除测试文件 | `file_delete` | 清理无效下载物 |

## 5. 决策规则

| 情况 | 处理策略 |
|------|----------|
| `all` 分类报错或不可用 | 改按分类逐个查询 |
| 用户只问“有什么” | 先概览，不主动下载 |
| 下载成功但内容明显是测试 JSON | 明确提示不可直接使用 |
| 用户要求删除测试文件 | 删除并回报路径与结果 |
| 工坊暂无可用作品 | 如实说明，不虚构正式可用能力 |

## 6. 失败经验

| 时间 | 问题 | 原因 | 解决方案 |
|------|------|------|----------|
| 2026-03-09 | 试图用统一分类直接总览失败 | 当前工坊接口不支持或当前环境不接受 `all` 分类 | 改为 `skills / methods / plugins / facets / mcp / apps` 逐分类查询 |
| 2026-03-09 | facet 文件容易被误判为可直接切换模组 | 下载结果只是测试 JSON，不是本地 FACET 规范文件 | 下载后必须读取或核验结构，再判断是否可用 |
| 2026-03-09 | 容易把工坊测试内容说成正式能力 | 只看名称，没有核验内容结构 | 必须结合下载文件内容和实际格式给结论 |

## 7. 成功案例

### 案例 1：浏览工坊并按分类给出总览（2026-03-09）
- **任务**：用户要求“到工坊里看看有些什么”
- **步骤**：
  1. 校验 UUID
  2. 使用 `office_workshop_search` 逐分类浏览
  3. 汇总各分类的可见作品
  4. 标记 apps 分类暂无结果
- **结果**：成功输出工坊内容概览，并说明统一分类查询失败后改为分类查询
- **结论**：逐分类查询是当前环境下更稳妥的工坊浏览方式

### 案例 2：下载“全栈翻译官”并判定为测试文件（2026-03-09）
- **任务**：下载工坊作品“全栈翻译官”
- **步骤**：
  1. 搜索并定位 facet 类目标作品
  2. 使用 `office_workshop_download` 下载至 `downloads/office`
  3. 读取文件内容核验结构
  4. 根据内容判断其为测试 JSON
  5. 按用户要求删除测试文件
- **结果**：成功完成下载、核验、删除闭环
- **结论**：下载并不等于可用，工坊作品必须以实际文件结构为准
