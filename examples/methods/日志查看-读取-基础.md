## 工具选择

### log_read - 读取日志文件

**适用场景**：读取指定日期的日志文件，查看详细日志内容

| 维度 | 说明 |
|------|------|
| **优点** | 可查看完整日志内容，支持关键词过滤和级别过滤 |
| **缺点** | 需要知道具体日期，无搜索功能 |
| **评分** | ⭐⭐⭐⭐⭐ 必备工具 |

**参数说明**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `date` | string | 否 | 日期，格式 YYYY-MM-DD，默认为今天 |
| `keyword` | string | 否 | 关键词过滤，仅返回包含该关键词的行 |
| `level` | string | 否 | 过滤日志级别：debug/info/warn/error |
| `module` | string | 否 | 过滤模块名，如 gateway/agent/tools/memory/mcp/heartbeat |
| `tail` | number | 否 | 只返回最后 N 行，默认 100 |

**使用示例**：

```bash
# 查看今天的完整日志（默认）
log_read

# 查看指定日期的日志
log_read date="2026-02-19"

# 只看错误级别
log_read level="error"

# 只看最后 50 行
log_read tail=50

# 关键词过滤 + 级别过滤
log_read level="warn" keyword="MCP"

# 特定模块
log_read module="gateway"
```

### log_search - 搜索日志

**适用场景**：在日志中搜索特定关键词、错误信息

| 维度 | 说明 |
|------|------|
| **优点** | 支持搜索多个日期范围，可返回匹配行 |
| **缺点** | **必须填写 query 参数，否则会失败** |
| **评分** | ⭐⭐⭐⭐⭐ 必备工具 |

**参数说明**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `query` | string | **是** | 搜索关键词（支持大小写敏感的子串匹配） |
| `startDate` | string | 否 | 开始日期 YYYY-MM-DD，默认最近 3 天 |
| `endDate` | string | 否 | 结束日期 YYYY-MM-DD，默认今天 |
| `level` | string | 否 | 过滤日志级别：debug/info/warn/error |
| `maxLines` | number | 否 | 最多返回匹配行数，默认 200 |

**使用示例**：

```bash
# 搜索错误关键词
log_search query="error"

# 搜索特定错误信息
log_search query="Connection refused"

# 搜索最近 7 天的日志
log_search query="MCP" startDate="2026-02-13" endDate="2026-02-20"

# 只看 error 级别的
log_search query="failed" level="error"

# 限制返回行数
log_search query="warning" maxLines=50
```

## 执行步骤

### 场景 1：日常健康检查

```
1. 使用 log_read（无参数）查看今天日志
2. 确认 gateway、MCP 连接状态
3. 检查是否有 error 级别日志
4. 如有警告，评估是否需要处理
```

### 场景 2：问题排查

```
1. 使用 log_search 搜索错误关键词
2. 缩小范围：添加日期、级别、模块过滤
3. 使用 log_read 查看具体日志详情
4. 根据错误信息定位问题
```

### 场景 3：查看历史操作

```
1. 使用 log_search 搜索相关关键词
2. 添加日期范围过滤
3. 查看具体操作记录和时间戳
```

## 失败经验

| 时间 | 问题 | 原因 | 解决方案 |
|------|------|------|----------|
| 2026-02-20 | log_search 调用失败 | 传入空 query 参数 | **query 是必填参数，必须提供搜索关键词** |
| 2026-02-20 | log_read 用了不存在的日期 | 日期格式错误或那天没有日志 | 使用正确格式 YYYY-MM-DD，检查日期是否存在 |
