## 当前推荐流程（Windows 正式版）

### 0. 推荐策略
拆成两个独立阶段：
1. **截图阶段**：优先用 Windows PowerShell 原生命令，保证稳定成功
2. **分析阶段**：通过 `.env` 读取 `OPENAI_API_KEY` / `OPENAI_BASE_URL` / `OPENAI_VISION_MODEL`

### 1. 正式入口文件
#### A. `skills/screen-capture/scripts/screen-capture.ps1`
职责：
- 接收 `OutputDir` / `Name` / `MultiMonitor`
- 执行截图
- 返回图片完整路径

**v2.2 整洁化增强：**
- 增加脚本文档说明
- 自动补 `.png` 后缀
- 对目录创建、屏幕区域、保存流程增加错误提示

#### B. `skills/screen-capture/scripts/screen-capture-analyze.ps1`
职责：

#### C. `skills/screen-capture/scripts/media-analyze-openai.py`

- 自动加载 `.env`
- 读取 `OPENAI_API_KEY`

- 压缩超大图片后调用视觉模型
- 输出 JSON（`description` / `tags` / `content`）

**v2.2 整洁化增强：**
- 拆出 `validate_env()` 与 `analyze_image()`
- 增强 API 认证失败 / 配置缺失 / 文件异常等错误提示

## 技术实现

### 当前主路径：Windows 原生截图
实际跑通的路径：
1. 用 PowerShell 加载 `System.Windows.Forms` 和 `System.Drawing`
2. 获取虚拟屏幕尺寸（兼容多屏）
3. 复制屏幕像素到位图
4. 保存为 PNG 到 `C:\Users\admin\Pictures\Screenshots`

### 当前分析路径：OpenAI 兼容视觉接口
1. `screen-capture-analyze.ps1` 读取 `.env`
2. 若未指定输入图，则先截图
3. 调用 `media-analyze-openai.py`
4. Python 读取 `OPENAI_API_KEY` / `OPENAI_BASE_URL` / `OPENAI_VISION_MODEL`
5. 若图片过大则压缩
6. 发送 `chat.completions.create`
7. 输出标准 JSON

## 失败经验

### 当前环境复核新增（2026-03-08）
| 错误场景 | 错误信息 | 原因 | 解决 |
|---------|---------|------|------|
| 无法直接执行 `.sh` | `Security Error: Command 'bash' is not in the safe list.` | 当前命令执行环境对 `bash` 有白名单限制 | 改用 **PowerShell 原生命令** 作为主入口 |
| `.env` 已存在但脚本未生效 | 无显式报错，但分析脚本未吃到配置 | `screen-capture-analyze.ps1` 缺少 `-EnvFile` 和导入逻辑 | 增加 `Import-DotEnvFile`，在 PowerShell 进程内注入环境变量 |
| 配了 `OPENAI_BASE_URL` 仍未走兼容接口 | 无法切到 Moonshot/Kimi 兼容地址 | `media-analyze-openai.py` 初始化客户端时未传 `base_url` | 改为 `OpenAI(api_key=api_key, base_url=base_url)` |
| Gemini 旧脚本不宜复用 | 脚本内可见硬编码 API Key | 不符合安全规范 | 改写为读取环境变量 / `.env`，删除硬编码密钥 |
| 正式链路与旧链路混放 | 使用者容易误用 `.sh` 方案 | 目录结构没有反映主/旧优先级 | 将旧脚本迁移到 `scripts/legacy/`，在结构层面降级旧链路 |
| 使用 `if not exist` 创建目录失败 | `Security Error: Command 'if' is not in the safe list.` | 当前命令白名单对 `if` 不放行 | 先直接执行文件移动；若目标目录已存在，则可继续完成迁移 |

### 历史失败经验（保留）
| 错误场景 | 错误信息 | 原因 | 解决 |
|---------|---------|------|------|
| GDI+ 错误 | `A generic error occurred in GDI+` | 目录不存在或无权限 | 先创建目录 |
| 路径错误 | `UNC paths are not supported` | CMD 不支持 WSL 的 UNC 路径 | 先切换到合适路径再执行 |
| API 连接失败 | `fetch failed` / `Connection error` | 外网访问或代理问题 | 配置网络代理，或改用当前可访问模型 |
